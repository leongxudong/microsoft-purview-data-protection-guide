# Microsoft Purview Data Protection Guide

A lightweight portfolio project documenting practical Microsoft Purview data protection concepts, including sensitivity labels, DLP policy design, testing, and rollout planning.

## Project Purpose

The purpose of this repository is to show how data protection controls can be planned and explained in a Microsoft-native environment.

## Scope

This project covers:

- Data classification model
- Sensitivity label design
- DLP policy design
- Testing and rollout approach
- Screenshot placeholders for Purview evidence

## Control Areas

| Area | Purpose |
|---|---|
| Data classification | Define how information should be grouped by sensitivity |
| Sensitivity labels | Apply visual markings and protection settings |
| DLP policies | Detect and control risky data sharing |
| User notifications | Inform users at the point of action |
| Testing | Validate impact before enforcement |
| Rollout | Phase controls to reduce operational disruption |

## Portfolio Value

This project demonstrates:

- Data protection governance
- Microsoft Purview familiarity
- Policy design thinking
- Compliance and user impact awareness
- Practical rollout planning
