# Sensitivity Labels

Sensitivity labels help users classify and protect information based on business sensitivity.

## Example Label Taxonomy

| Label | Intended Use | Possible Controls |
|---|---|---|
| Public | Information approved for public release | No encryption required |
| Internal | General internal information | Visual marking optional |
| Confidential | Sensitive business or personal information | Marking, restricted sharing, possible encryption |
| Restricted | Highly sensitive or regulated information | Strong access restriction and monitoring |

## Label Design Principles

- Keep the number of labels manageable.
- Use names that users can understand.
- Avoid over-classification by default.
- Define clear examples for each label.
- Test user experience before enforcement.

## Policy Questions

- Should users apply labels manually?
- Should default labels be used?
- Should auto-labeling be introduced later?
- Should labels apply encryption?
- How should external sharing be handled?
